# 周报生成器

自动分析当前git分支本周提交记录，生成结构化周报。

## 使用方法

用户说"生成周报"或"周报"时触发。

## 执行流程

### 1. 获取本周git提交记录

```bash
git log --since="1 week ago" --pretty=format:"%h - %s" --author="$(git config user.name)"
```

### 2. 生成周报内容

**格式要求：**
- 自动汇总为10条
- 每条20-30字
- 提交信息过短则补充技术细节
- 提交信息过长则精简核心内容

**输出格式：**
```
本周工作汇报（共N条提交）

1. [第一条工作内容]

2. [第二条工作内容]

...

10. [第十条工作内容]
```

### 3. 内容处理规则

- **去重**：相同功能合并为一条
- **分类**：按功能模块归类（如：前端、后端、配置、文档）
- **排序**：按提交时间倒序，最新的在前
- **精简**：过滤无意义提交（如：merge、wip、tmp）

## 示例

输入：
```
生成周报
```

输出：
```
本周工作汇报（共15条提交）

1. 完成用户登录功能开发，支持邮箱和手机号登录

6. 重构用户模块代码，提升代码可维护性

```
